# 美赛团队任务清单 (MCM/ICM)

> **目标**: 明确分工，避免手忙脚乱，高效协作完成比赛

---

## 团队角色定义

| 角色 | 主要职责 | 技能要求 |
|------|----------|----------|
| **建模手** | 问题分析、模型设计、数学推导、论文主体撰写 | 数学建模、论文写作 |
| **编程手** | 数据处理、算法实现、可视化、代码调试 | Python/MATLAB、数据分析 |
| **论文手** | 论文排版、图表整理、摘要撰写、格式检查 | LaTeX/Word、英文写作 |

> 注：三人团队通常为 建模+编程+论文，或 建模+编程(兼论文)

---

## 比赛时间线 (4天96小时)

```
Day 1 (0-24h)   : 读题 → 数据探索 → 确定方向 → 初步建模
Day 2 (24-48h)  : 核心模型开发 → 代码实现 → 初步结果
Day 3 (48-72h)  : 模型优化 → 敏感性分析 → 论文框架
Day 4 (72-96h)  : 论文撰写 → 图表美化 → 检查提交
```

## 九、紧急情况处理

### 常见问题及解决方案

| 问题 | 解决方案 | 负责人 |
|------|----------|--------|
| 模型效果差 | 简化模型/换方法/调参数 | 建模+编程 |
| 代码报错 | 检查数据/简化代码/求助 | 编程手 |
| 时间不够 | 砍掉次要内容/分工并行 | 全队 |
| 结果不合理 | 检查数据/检查公式/重新运行 | 编程手 |
| 论文写不完 | 先完成框架/后填充细节 | 建模手 |

### 时间紧急时的优先级

```
1. 确保所有子问题有答案 (即使简单)
2. 确保有基本的模型和结果
3. 确保摘要完整
4. 确保格式符合要求
5. 再考虑优化和美化
```
